# Specs

## Formato

Criar arquivo `specs/<feature-name>.md` com:

```markdown
# <Feature Name>

## Conclusão da Pesquisa
- Alternativas avaliadas
- Biblioteca/ferramenta recomendada

## Especificação
- Requisitos funcionais
- Fluxo do usuário

## To-Dos
- [ ] TODO 1
- [ ] TODO 2

## Perguntas
- ❓ Pergunta para stakeholder
```

## Processo

1. Criar spec ANTES de implementar
2. Pesquisar alternativas e documentar
3. Fazer perguntas ao stakeholder
4. Implementar após aprovação
